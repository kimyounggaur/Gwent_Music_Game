# 『앙상블 클래시: 솔로』 — 1인용(vs AI) 바이브코딩 프롬프트북

> 위쳐3 '궨트' 시스템을 음악이론에 이식한 **1인용 카드배틀 웹앱**을, 코딩 에이전트(Claude Code, Cursor 등)에 순서대로 던질 수 있는 프롬프트로 완성하는 문서입니다. **이 파일 하나만 있으면 외부 문서 없이 개발 가능**하도록 룰 요약·AI 명세·카드 시드를 모두 내장했습니다.

---

## 0. 사용법 (반드시 먼저 읽기)

1. **P1부터 순서대로** 하나씩 실행합니다. 각 프롬프트 끝의 `완료 기준`을 직접 확인한 뒤에만 다음으로 넘어갑니다.
2. 새 에이전트 세션을 열 때마다 **§4 공통 컨텍스트 블록을 먼저 붙여넣고** 그다음 해당 프롬프트를 붙여넣습니다.
3. **회귀 규칙:** "기존 테스트가 깨지면 임의로 테스트를 수정하지 말고 멈춰서 보고할 것"을 에이전트에게 항상 요구합니다.
4. 프롬프트 1개 완료 = git 커밋 1개. 롤백 가능 상태를 유지합니다.
5. P1~P9까지만 해도 **AI와 한 판을 끝까지 플레이**할 수 있습니다. 이후는 카덴차·청음·학습 루프·캠페인 순으로 살이 붙습니다.

## 1. 제품 정의 (확정 사양)

**한 줄:** 멜로디·화성·리듬 3개 행에 카드를 배치해 점수 합으로 악장을 따내는 3악장 2선승 카드배틀. 상대는 **AI 지휘자**(3난이도). 강한 카드 효과는 **음악이론·청음 문제(카덴차 체크)** 를 맞혀야 발동.

### 1.1 핵심 룰 (위쳐3식 고정)

| 항목 | 사양 |
|---|---|
| 승리 | 3악장 중 2악장 선취. 동점 악장은 양쪽 모두 승점(둘 다 2승점이면 무승부) |
| 손패 | 시작 시 **10장**, 멀리건 **2장**, **악장 간 드로우 없음** (위쳐3 그대로 — 자원관리가 곧 게임) |
| 턴 | 번갈아 카드 1장 플레이 또는 **패스**. 패스하면 그 악장에서 행동 종료(상대는 계속 가능). 큐(Order)·지휘자 능력은 카드 플레이 전후 자유 사용 |
| 전장 | 양쪽 각자 **멜로디 / 화성 / 리듬** 3개 행. 행당 최대 7장 |
| 악장 종료 | 양쪽 다 패스 → 행 점수 총합 높은 쪽 승점 1. 전장 카드는 보관함行(지속음 보유 카드는 잔류) |
| 카덴차(2겹 구조) | 모든 유닛: ① **기본 파워**(무조건 적용) ② **카덴차 효과**(체크 통과 시에만). 문제와 보상은 **내기 전에 미리 보여줌** → 운빨 퀴즈가 아닌 계산된 베팅 |
| 방해 기호(날씨) | 쉼표→리듬 / 약음기→멜로디 / 불협화음→화성. 대상 행의 비면역 유닛 **기본 파워를 1로 간주**(증폭·약화는 그 위에 적용). 투티 클리어로 해제. 명인(골드)은 면역 |
| 선공 | 동전 던지기. 선공 보정(선제 악상)은 MVP 제외 — 추후 옵션 |

### 1.2 점수 계산 순서 (엔진 규약 — 모든 프롬프트가 이 순서를 따름)

```
1. base        = 방해 기호 적용 행이고 비면역이면 1, 아니면 카드 기본 파워
2. raw         = base + boost − damage      (0 이하면 즉시 사망 → 보관함)
3. ensemble    = 화음 결속(구성음 완성 시 멤버 ×2) → 종지 인접 보너스(+N) → 조성 결속(동일 조표 3장↑ 행 전체 +1)
4. fortissimo  = 해당 행에 포르티시모 활성 시 비명인 유닛 ×2 (마지막에 적용)
```

### 1.3 MVP 범위

| 포함 (P1~P12) | 제외/후순위 |
|---|---|
| vs AI 1인 플레이 전체 루프 | 온라인 2인전, Realtime |
| 카덴차 이론형 + 청음형(Tone.js) | 서버 채점(싱글이라 클라 채점 — §2.3) |
| 진영 2~3개 스타터 덱, AI 덱 3종 | 6진영 전체, 드래프트 모드 |
| localStorage 영속화 | Supabase(P15에서 도입) |
| — | 선제 악상, 입문 모드(8장+드로우) 토글 |

## 2. 기술 결정

### 2.1 스택

- **Next.js (App Router) + TypeScript + Tailwind** — UI
- **Zustand** — 클라이언트 상태 (게임 상태는 엔진이 소유, 스토어는 래핑만)
- **순수 함수 게임 엔진** (`src/engine/`) — UI·스토어와 완전 분리, **시드 RNG**로 결정적 재현(테스트·리플레이·AI 시뮬레이션의 전제)
- **Vitest** — 엔진·AI 단위/시뮬레이션 테스트
- **Tone.js** — 청음 문제 생성·재생 (상업 음원 불사용 → 저작권 안전, 문제 무한 생성)
- **localStorage → Supabase**(P15) — 덱·학습 데이터·진행도

### 2.2 폴더 구조 (P1에서 고정)

```
src/
  engine/        # 순수 함수: 상태, 수 적용, 점수, 시너지, 효과 (React import 금지)
  ai/            # AI 지휘자: 수 생성, 평가, 패스 결정, 페르소나
  audio/         # Tone.js 래퍼: 음정/화음/리듬 생성·재생
  cadenza/       # 문제 출제·채점 (이론형/청음형)
  data/          # cards.ts (카드 시드), decks.ts (스타터/AI 덱)
  store/         # Zustand
  components/    # UI
  app/           # Next.js 라우트
```

### 2.3 채점·치팅 정책 (1인용의 단순화)

싱글플레이어는 치팅 피해자가 본인뿐이므로 **MVP는 클라이언트 채점**으로 갑니다(개발 속도 우선). 단, 정답 데이터는 별도 모듈(`cadenza/answers.ts`)에 격리해 두어 추후 랭크/리더보드 도입 시 **그 모듈만 Supabase Edge Function으로 이전**하면 서버 채점으로 전환되도록 구조를 잡습니다(P15에 마이그레이션 경로 포함).

## 3. AI 지휘자 설계 명세 ★(이 문서의 새 심장)

궨트의 재미는 점수가 아니라 **"언제 포기하느냐"의 심리전**입니다. AI가 이걸 못 하면 게임이 죽습니다. 아래 명세는 P6~P8 프롬프트가 그대로 구현합니다.

### 3.1 난이도 페르소나

| | 견습 지휘자 (Easy) | 단원장 (Normal) | 마에스트로 (Hard) |
|---|---|---|---|
| 수 선택 | 그리디(즉시 점수 최대) + 30% 무작위 | 평가 함수 풀가동 | 평가 함수 + 1수 앞 가정(상대 최선 응수 추정) |
| 패스 성향 | 손패 아끼지 않음, 단순 임계값 | §3.3 결정 트리 전체 | 결정 트리 + 미끼 패스(이기는 중 일부러 카드 1장 더 유도) |
| 명인 온존 | 안 함 | 1악장에 명인 최대 1장 | 우세 시 명인 0장, 3악장 몰빵 |
| 카덴차 성공률 (Tier I/II/III/IV) | 0.50 / 0.30 / 0.15 / 0.05 | 0.75 / 0.60 / 0.45 / 0.25 | 0.95 / 0.85 / 0.70 / 0.55 |

> **AI의 카덴차 = 확률 굴림.** "AI 지휘자가 카덴차에 도전합니다…" 연출(0.8~1.5초) 후 성공/실패 표시. 난이도가 곧 AI의 '이론 실력'이라는 세계관 일치 — 플레이어와 같은 룰로 싸우는 느낌을 줍니다.

### 3.2 턴 파이프라인

```
1. 상태 평가: round, victoryTokens, 내/상대 점수, 내/상대 남은 손패 수, 상대 패스 여부
2. PASS 판단 (§3.3) — 패스면 종료
3. 후보 수 생성: 손패 × 합법 행 + 큐/리더 사용 조합
4. 각 후보를 §3.4 평가 함수로 점수화 → 최고점 선택 (Easy는 상위 3개 중 무작위)
5. 카드에 카덴차가 있으면 §3.1 확률로 도전 굴림 → 결과 반영
```

### 3.3 패스 결정 트리 (그대로 코드화할 것)

```ts
function shouldPass(s: AiView): boolean {
  const lead = s.myScore - s.oppScore;
  const mustWin = s.myTokens === 0 && s.oppTokens === 1;   // 지면 패배 확정인 악장

  if (s.oppPassed) {
    if (lead > 0) return true;                              // 이미 이김 → 굳히기 패스
    if (-lead > maxSwing(s.myHand)) return true;            // 남은 손패로 역전 불가 → 손절
    return false;                                           // 최소 카드로 +1만 넘기고 다음 턴 패스 (평가 함수가 처리)
  }
  if (mustWin) return s.myHand.length === 0;                // 배수진: 빈손 전까지 절대 패스 금지
  if (s.round === 1) {
    // 카드 어드밴티지 교환: 크게 지는 중 + 내가 카드를 덜 썼으면 악장을 내준다
    if (-lead >= 12 && s.cardsSpentDiff <= -1) return true;
    if (-lead >= 20) return true;
  }
  if (s.round >= 2 && s.myTokens === 1 && lead > 0 && s.oppHand <= 1) return true; // 상대 빈손 직전 + 우세 → 마무리
  return false;
}
// maxSwing(hand) = Σ(기본파워) + Σ(카덴차 기대값: 성공률 × 보상치)
```

### 3.4 수 평가 함수 (시작 가중치 — P8 시뮬레이션으로 튜닝)

```
score(move) =
    immediatePoints                          // 기본 파워 + 즉시 효과(스타카토 제거치 등)
  + cadenzaEV                                // successProb(tier, 난이도) × 카덴차 보상 추정치
  + synergyDelta                             // 화음 결속 완성 +8 / 한 장 남음 +3, 종지 셋업 +2, 조성 결속 기여 +1
  + interferenceValue                        // 방해 기호: 상대 대상 행 (현재합 − 유닛수×1) × 0.8, 단 상대 패스 후엔 ×1.2
  − weatherRisk                              // 내 방해 걸린 행에 비면역 유닛 배치 시 −(기본파워−1)
  − holdPenalty                              // 1악장에 명인 내기: 우세 중이면 −6, 열세면 0 (난이도별 §3.1)
  − overcommit                               // 상대 패스 후 필요 이상 점수 초과분 × 0.5
```

### 3.5 AI 덱

난이도별 고정 덱 3종(P2에서 정의): 견습=리듬 연구소(직선적), 단원장=단조 길드(보관함 재활용), 마에스트로=재즈 화성 클럽(텐션·고위험 고수익 — 높은 카덴차 성공률과 시너지).

## 4. 공통 컨텍스트 블록 (매 세션 시작 시 복붙)

```
[프로젝트 컨텍스트 — 항상 이 규약을 따를 것]
제품: 『앙상블 클래시: 솔로』 — 궨트식 1인용(vs AI) 음악이론 카드배틀 웹앱.
스택: Next.js(App Router)+TypeScript+Tailwind, Zustand, Vitest, Tone.js. 게임 로직은 src/engine/의 순수 함수(React import 금지), AI는 src/ai/, 오디오는 src/audio/, 출제·채점은 src/cadenza/.

핵심 룰:
- 3악장 2선승. 동점 악장은 양쪽 승점. 손패 10장, 멀리건 2, 악장 간 드로우 없음.
- 행: melody/harmony/rhythm, 행당 최대 7장. 턴마다 카드 1장 또는 패스(패스=악장 이탈, 상대는 계속).
- 유닛 = 기본 파워(무조건) + 카덴차(이론/청음 체크 통과 시 강효과). 문제·보상은 플레이 전 미리보기.
- 방해 기호: 쉼표→rhythm, 약음기→melody, 불협화음→harmony. 대상 행 비면역 유닛의 base를 1로 간주, boost/damage는 그 위에 적용. 명인(virtuoso)은 면역. 투티 클리어로 전부 해제.
- 점수 순서: ①weathered base ②+boost−damage(≤0 사망) ③화음결속(멤버 전원 같은 행이면 각 ×2)→종지 인접(+N)→조성결속(동일 key 태그 3장↑ 행 전체 +1) ④포르티시모(행의 비명인 ×2, 마지막).
- RNG는 반드시 시드 기반(createRng(seed)) — 셔플·AI 굴림 전부. Math.random 직접 호출 금지.

작업 규약:
- 기존 테스트가 깨지면 테스트를 고치지 말고 멈춰서 원인을 보고할 것.
- 모든 엔진/AI 함수는 불변(immutable) 업데이트. 새 GameState를 반환.
- 타입은 src/engine/types.ts의 정의를 단일 진실로 사용. 임의 변경 금지(변경 필요 시 사유 보고).
```

## 5. 단계별 프롬프트

### Phase A — 기반

#### P1. 프로젝트 스캐폴딩 + 타입 시스템

```
Next.js(App Router)+TypeScript+Tailwind+Zustand+Vitest 프로젝트를 만들어줘. §공통 컨텍스트의 폴더 구조를 생성하고, src/engine/types.ts에 아래 타입을 정의해.

type RowId = 'melody'|'harmony'|'rhythm';
type Side = 'player'|'ai';
type Faction = 'maggiore'|'minore'|'tempo'|'tension'|'neutral';
type CardType = 'bronze'|'virtuoso'|'special'|'leader';
type ChallengeKind = 'theory_mc'|'theory_build'|'ear_interval'|'ear_chord'|'ear_rhythm';

interface EffectSpec { kind:'boost_self'|'boost_adjacent'|'boost_next_tag'|'boost_row'|'scorch'|'summon'|'multiply_row'|'draw'; amount?:number; tag?:string; row?:RowId; }
interface CadenzaSpec { tier:1|2|3|4; kind:ChallengeKind; effect:EffectSpec; }
interface EnsembleSpec { bondGroup?:string; bondMembers?:string[]; cadence?:{ withTag:string; bonus:number }; keyTag?:string; }
interface CardDef { id:string; nameKo:string; faction:Faction; row:RowId|'any'; type:CardType; basePower:number|null; tags:string[]; cadenza?:CadenzaSpec; ensemble?:EnsembleSpec; immune?:boolean; special?:EffectSpec|{kind:'weather';row:RowId}|{kind:'clear'}|{kind:'fortissimo'}|{kind:'reprise'}|{kind:'fermata'}; }
interface UnitInstance { uid:string; cardId:string; boost:number; damage:number; sustain:boolean; cadenzaDone:boolean; }
interface PlayerState { hand:string[]; deck:string[]; archive:string[]; rows:Record<RowId,UnitInstance[]>; passed:boolean; tokens:number; mulligansLeft:number; leaderId:string; leaderCharges:number; fortissimo:Partial<Record<RowId,boolean>>; }
interface GameState { seed:string; rngState:number; round:1|2|3; turn:Side; phase:'mulligan'|'play'|'roundEnd'|'gameEnd'; players:Record<Side,PlayerState>; weather:Record<Side,Partial<Record<RowId,boolean>>>; winner:Side|'draw'|null; log:GameEvent[]; }
type Move = {kind:'play'; cardUid:string; row:RowId; cadenzaSuccess?:boolean} | {kind:'pass'} | {kind:'leader'} | {kind:'mulligan'; cardUid:string} | {kind:'mulliganDone'};

src/engine/rng.ts에 mulberry32 기반 createRng(seed:string)과 shuffle(arr, rng)을 만들고 테스트 작성(같은 시드 → 같은 셔플).

완료 기준: pnpm build 성공, pnpm test 통과(rng 테스트 포함), 빈 홈페이지 렌더.
```

#### P2. 카드 시드 30종 + 덱 정의

음색: 깨끗한 Synth(triangle)+짧은 release. 난이도 파라미터: 음역(C3~C5/넓게), bpm(70~120), 동시발음 여부.

테스트 페이지 app/audio-test/page.tsx: 각 API를 버튼으로 수동 검증(장3도/단3도/완전5도, maj/min/dom7/C9, 셋잇단 패턴 등).

완료 기준: 테스트 페이지에서 모든 소리가 명확히 구분되어 재생, 모바일 사파리에서도 재생(제스처 후), 연속 재생 시 소리 겹침 없음(stopAll 동작).
```

#### P12. 청음 카덴차 3종 통합

```
P10의 카덴차 플로우에 청음형 3종을 추가해줘. 공통: 듣기 버튼(최대 3회 재청취, 남은 횟수 표시), 문제 파라미터는 시드RNG로 생성하되 정답 라벨은 answers.ts 경유.

1) ear_interval: '재생된 음정은?' 4지선다. T1=장3/단3/완전5 멜로딕, T2=2~6도 혼합, T3=하모닉+감/증 포함.
2) ear_chord: '재생된 화음은?' T1=장/단 2지, T2=장/단/감/dom7 4지, T3=텐션·전위 포함 4지.
3) ear_rhythm: 기본=패턴 4지선다(악보 대신 블록 표기 ▮▮▯▮ + 점/셋잇단 기호). 보너스: '탭 따라치기' 모드 — 스페이스/터치 탭 타임스탬프를 16분음표 그리드로 양자화, 각 온셋 오차 ±90ms 이내 80%↑면 정답.

완료 기준: 청음 카덴차 카드(완전5도, 셋잇단, C9 등)가 실제 소리로 출제·채점되고 보상이 적용. 재청취 3회 제한 동작. 폴백 설정 동작.

## 7. 트러블슈팅 팁

- **AI가 이상한 수를 둘 때:** `runGame` 로그(`state.log`)를 시드와 함께 저장해 재현 → 평가 함수 가중치(AI_WEIGHTS)를 의심하기 전에 `legalMoves`·`shouldPass` 입력값(view)을 먼저 검증.
- **점수가 안 맞을 때:** §1.2 순서 위반이 대부분. 결속 ×2가 boost 이전에 적용되거나, 포르티시모가 명인에 적용되는 버그가 단골.
- **Tone.js 무음:** 사용자 제스처 전 `Tone.start()` 미호출. iOS는 음소거 스위치도 확인.
- **테스트 플레이키:** Math.random 직접 호출이 숨어든 것. 전부 시드RNG 경유인지 grep.
- **밸런스 감각:** 시뮬레이션 승률보다 먼저 "한 악장 총점 30~60, 평균 게임 16~22턴" 범위를 지키면 체감이 안정됩니다.
